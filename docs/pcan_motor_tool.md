# PCAN 电机调试上位机

## 1. 功能

`scripts/pcan_motor_tool.py` 用于通过 PCAN 直接调试 `0x01~0x08` 电机节点。

- 接收 `0x201~0x208` 电机主动上报帧。
- 在表格中显示位置、速度、版本号、上报配置、最后应答和发送计数。
- 按当前选中的电机绘制位置和速度曲线。
- 通过 `0x100 + node_id` 发送电机控制命令。
- 支持电流、速度、位置三种模式的使能和目标值发送。
- 支持配置或读取电机主动上报频率。
- 支持执行电机零点校准并读取零点值。
- 支持停止当前电机或停止全部电机输出。

## 2. 启动

默认使用 `PCAN_USBBUS1` 和 `1M` 波特率。

```powershell
python project/duck-mid-f103/scripts/pcan_motor_tool.py
```

指定通道和波特率：

```powershell
python project/duck-mid-f103/scripts/pcan_motor_tool.py --channel PCAN_USBBUS1 --bitrate 1M
```

如果连接时报 `The value of a handle ... is invalid`，通常是 PCAN 通道选择不对。可以点击界面上的“扫描”，工具会尝试打开 `PCAN_USBBUS1~PCAN_USBBUS16` 并列出可用通道。

## 4. 控制

模式：

| 模式 | 编号 | 目标单位 |
| --- | --- | --- |
| `current` | `0` | 电机电流控制量 |
| `speed` | `1` | `mrad/s` |
| `position` | `2` | `mrad` |

常用操作：

- “设置模式”：发送 `0x01`，包含使能状态和控制模式。
- “发送目标”：按当前模式发送 `0x02`、`0x03` 或 `0x04`。
- “停止当前”：先发送电流目标 `0`，再关闭当前电机使能。
- “停止全部”：依次对 `0x01~0x08` 节点执行停止输出。
- “读取全部”：读取节点号、版本号、上报配置和零点。
- “零点校准”：发送 `0x30`，电机按当前机械位置执行零点校准并保存，成功应答返回零点值。
- “读取零点”：发送 `0x31`，成功应答返回当前零点值。

## 5. 曲线

曲线区显示当前“电机”下拉框选中的节点。

- 蓝色曲线是位置，单位 `mrad`。
- 橙色曲线是速度，单位 `mrad/s`。
- “窗口 s”用于设置参与绘制的最近数据长度，范围 `0.2~120s`。

## 6. 上报频率

上报配置使用命令 `0x23`，读取上报配置使用命令 `0x24`。

工具按输入的 Hz 换算为周期：

```text
period_ms = round(1000 / hz)
```

当前工具限制周期范围为 `2~60000ms`，所以最高建议配置为 `500Hz`。
