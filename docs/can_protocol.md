## 1. 概述

本项目使用标准帧 `CAN 2.0A`。除特别说明外，数据长度固定按 8 字节处理；多字节整数统一使用小端格式。

每台电机出厂或擦除参数后处于未配置状态：

- 管理节点 ID：`0x7E`
- 管理命令 ID：`0x17E`
- 管理应答 ID：`0x1FE`
- 管理上报 ID：`0x27E`
- 未配置电机以 2Hz 频率上报 `UID32`
- 未配置电机不响应普通控制命令，不主动上报电机状态，也不允许 OTA
- 配置业务 CAN ID 后进入运行状态，并使用业务节点 ID 响应控制命令

`UID32` 是由 STM32F103 96 位 UID 派生出的 32 位短 UID，用于在管理 ID 下区分同总线上的多台电机。

## 2. ID 规划

标准帧 ID 由基础 ID 与节点 ID 相加得到：

| 类型 | 标准帧 ID |
| --- | --- |
| 主机命令 | `0x100 + node_id` |
| 电机应答 | `0x180 + node_id` |
| 电机上报 | `0x200 + node_id` |
| 云台目标预留 | `0x300` |
| OTA 控制 | `0x400 + node_id` |
| OTA 响应 | `0x500 + node_id` |

管理节点固定使用 `node_id = 0x7E`。

## 3. 通用帧格式

| 字节 | 含义 |
| --- | --- |
| Byte0 | 命令码 |
| Byte1~Byte7 | 命令参数 |

电机主动上报帧不使用该通用命令格式，具体格式见第 8 节。

应答帧格式：

| 字节 | 含义 |
| --- | --- |
| Byte0 | 回显命令码 |
| Byte1 | 状态码 |
| Byte2~Byte7 | 附加返回数据 |

状态码：

| 状态码 | 含义 |
| --- | --- |
| `0x00` | 成功 |
| `0x01` | 命令非法 |
| `0x02` | 参数非法 |
| `0x03` | 模式非法 |
| `0x04` | CAN 错误 |

## 4. 管理命令

### 4.1 未配置发现上报

- 命令码：`0x60`
- 方向：电机 -> 主机
- 帧 ID：`0x27E`
- 频率：2Hz，仅未配置状态上报

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x60` |
| Byte1 | 配置状态，`0 = 未配置`，`1 = 已配置` |
| Byte2 | 当前节点 ID，未配置时为 `0x7E` |
| Byte3~Byte6 | `UID32`，小端 |
| Byte7 | 保留 |

### 4.2 识别灯效

- 命令码：`0x61`
- 方向：主机 -> 电机
- 帧 ID：`0x17E`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x61` |
| Byte1 | 闪烁频率，单位：Hz，固件限制到 1~20Hz |
| Byte2 | 持续时间，单位：秒，0 表示默认 5 秒，最大 60 秒 |
| Byte3~Byte6 | 目标 `UID32`，小端 |
| Byte7 | 保留 |

匹配 `UID32` 的电机会进入识别灯效，并通过 `0x1FE` 应答：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x61` |
| Byte1 | 状态码 |
| Byte2 | 实际闪烁频率，单位：Hz |
| Byte3 | 实际持续时间，单位：秒 |
| Byte4~Byte7 | `UID32`，小端 |

### 4.3 配置业务 CAN ID

- 命令码：`0x62`
- 方向：主机 -> 电机
- 帧 ID：`0x17E`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x62` |
| Byte1 | 新业务节点 ID，范围 `0x01~0x7F`，不能为 `0x7E` |
| Byte2 | 保留 |
| Byte3~Byte6 | 目标 `UID32`，小端 |
| Byte7 | 保留 |

匹配 `UID32` 且参数合法后，固件会保存 `can_id` 与 `can_configured = 1`，切换到运行状态，并重新配置 CAN 过滤器。

应答帧 ID 为 `0x1FE`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x62` |
| Byte1 | 状态码 |
| Byte2 | 新业务节点 ID |
| Byte3~Byte6 | `UID32`，小端 |
| Byte7 | 保留 |

## 5. 普通控制命令

普通控制命令只在电机已配置后生效。未配置状态下，即使收到普通控制命令也不会响应。

### 5.1 设置运行模式

- 命令码：`0x01`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x01` |
| Byte1 | 使能，`0 = 停机`，`1 = 运行` |
| Byte2 | 控制模式，`0 = 电流`，`1 = 速度`，`2 = 位置` |
| Byte3~Byte7 | 保留 |

### 5.2 设置电流目标

- 命令码：`0x02`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x02` |
| Byte1~Byte4 | 电流目标，`int32`，单位：q 轴内部控制量 |
| Byte5~Byte7 | 保留 |

### 5.3 设置速度目标

- 命令码：`0x03`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x03` |
| Byte1~Byte4 | 速度目标，`int32`，单位：mrad/s |
| Byte5~Byte7 | 保留 |

### 5.4 设置位置目标

- 命令码：`0x04`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x04` |
| Byte1~Byte4 | 位置目标，`int32`，单位：mrad |
| Byte5~Byte7 | 保留 |

## 6. PID 命令

PID 参数编号：

| 编号 | 含义 |
| --- | --- |
| `0x00` | `P` |
| `0x01` | `I` |
| `0x02` | `D` |
| `0x03` | `I_ACC_MAX` |
| `0x04` | `OUT_MAX` |

`P`、`I`、`D` 按 `x1000` 的整数传输，`I_ACC_MAX` 和 `OUT_MAX` 直接按内部整数值传输。

| 命令码 | 方向 | 含义 |
| --- | --- | --- |
| `0x10` | 主机 -> 电机 | 设置速度环参数 |
| `0x11` | 主机 -> 电机 | 读取速度环参数 |
| `0x12` | 主机 -> 电机 | 设置位置环参数 |
| `0x13` | 主机 -> 电机 | 读取位置环参数 |

设置参数帧：

| 字节 | 含义 |
| --- | --- |
| Byte0 | 命令码 |
| Byte1 | 参数编号 |
| Byte2~Byte5 | 参数值，`int32` |
| Byte6~Byte7 | 保留 |

读取参数帧：

| 字节 | 含义 |
| --- | --- |
| Byte0 | 命令码 |
| Byte1 | 参数编号 |
| Byte2~Byte7 | 保留 |

## 7. 参数与版本命令

| 命令码 | 含义 |
| --- | --- |
| `0x20` | 设置业务 CAN 节点 ID，配置后仍保持已配置状态 |
| `0x21` | 读取当前 CAN 节点 ID |
| `0x22` | 读取 App 版本号 |
| `0x23` | 设置电机主动上报配置 |
| `0x24` | 读取电机主动上报配置 |
| `0x25` | 复位 CAN 配置信息并重启 |
| `0x26` | 设置 FOC 基础配置 |
| `0x27` | 读取 FOC 基础配置 |
| `0x30` | 零点校准并保存 |
| `0x31` | 读取零点 |
| `0x40` | 进入 Boot OTA，仅已配置状态允许 |

读取 App 版本号应答负载：

| 字节 | 含义 |
| --- | --- |
| Byte2 | 主版本号 |
| Byte3 | 次版本号 |
| Byte4 | 修订版本号 |

当前初始版本为 `1.0.0`：

```text
DATA = 22 00 01 00 00 00 00 00
```

设置电机主动上报配置 `0x23`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x23` |
| Byte1 | 上报使能，`0 = 关闭`，`1 = 打开` |
| Byte2~Byte3 | 上报周期，`uint16`，单位：毫秒，范围 `1~60000` |
| Byte4~Byte7 | 保留 |

读取电机主动上报配置 `0x24`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x24` |
| Byte1~Byte7 | 保留 |

`0x23` 和 `0x24` 的应答负载：

| 字节 | 含义 |
| --- | --- |
| Byte2 | 上报使能，`0 = 关闭`，`1 = 打开` |
| Byte3~Byte4 | 上报周期，`uint16`，单位：毫秒 |
| Byte5~Byte7 | 保留 |

复位 CAN 配置信息 `0x25`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x25` |
| Byte1 | 确认码，固定为 `0xA5` |
| Byte2~Byte7 | 保留 |

该命令仅复位用户信息页中的 CAN ID 和 CAN 配置状态：`can_id` 恢复为管理节点 `0x7E`，`can_configured` 恢复为未配置。零点校准值、OTA 信息页、主动上报使能和主动上报周期保持不变。命令仅在已配置业务节点下生效，确认码错误时返回参数非法；应答成功后固件保存信息并软件复位。复位后电机回到未配置状态，使用管理节点 `0x7E` 上报 `UID32`。


设置 FOC 基础配置 `0x26`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x26` |
| Byte1 | 参数编号 |
| Byte2~Byte5 | 参数值，`uint32`，小端 |
| Byte6~Byte7 | 保留 |

读取 FOC 基础配置 `0x27`：

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x27` |
| Byte1 | 参数编号 |
| Byte2~Byte7 | 保留 |

`0x26` 和 `0x27` 的应答负载：

| 字节 | 含义 |
| --- | --- |
| Byte2 | 参数编号 |
| Byte3~Byte6 | 参数值，`uint32`，小端 |
| Byte7 | 保留 |

FOC 基础配置参数编号：

| 编号 | 参数 | 单位 | 默认值 | 数值范围 |
| --- | --- | --- | --- | --- |
| `0x00` | `pole_pairs` | 个 | `7` | `1~32` |
| `0x01` | `master_voltage_mv` | 毫伏 | `12000` | `1~60000` |
| `0x02` | `control_hz` | Hz | `2000` | `1~4000` |
| `0x03` | `sensor_hz` | Hz | `2000` | `1~4000` |
| `0x04` | `phase_map` | 无 | `0` | `0~5` |

固件只校验数值合法范围，不判断这些参数是否符合实际电机和电气系统。配置保存到用户信息页尾部扩展区，`0x25` 只复位 CAN ID 和 CAN 配置状态，不复位 FOC 基础配置。

`phase_map` 用于配置 FOC 逻辑三相 `A/B/C` 到 TIM1 物理输出 `CH1/CH2/CH3` 的连接顺序：

| `phase_map` | TIM1 CH1 | TIM1 CH2 | TIM1 CH3 |
| --- | --- | --- | --- |
| `0` | A | B | C |
| `1` | A | C | B |
| `2` | B | A | C |
| `3` | B | C | A |
| `4` | C | A | B |
| `5` | C | B | A |

## 8. 电机主动上报

电机主动上报默认关闭，默认周期为 `10ms`。已配置电机只有在 `0x23` 打开主动上报后，才会按配置周期上报位置和速度：

| 字节 | 含义 |
| --- | --- |
| Byte0~Byte3 | 当前位置，`int32`，单位：mrad |
| Byte4~Byte7 | 当前速度，`int32`，单位：mrad/s |

未配置电机只发送管理发现上报，不发送位置和速度上报。

## 9. 云台目标帧

云台目标帧用于主机或其他 CAN 节点向 mid 端下发目标，不直接发送给电机。当前 `gimbal` 核心模块不直接依赖 `can-protocol`，仅通过 `gen-motor` 控制电机；若后续需要 CAN 下发云台目标，应在独立的 CAN 业务模块中解析本节帧格式，再调用 `gimbal` 接口。

- 帧 ID：`0x300`
- 方向：主机/CAN 节点 -> mid
- 数据长度：`6~8` 字节

### 9.1 设置单轴目标

- 命令码：`0x01`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x01` |
| Byte1 | 轴编号，`0 = Roll`，`1 = Pitch`，`2 = Yaw` |
| Byte2~Byte5 | 目标角度，`int32`，单位：mrad |
| Byte6~Byte7 | 保留 |

Roll 和 Pitch 的目标值使用姿态坐标系，位置反馈来自 RK 串口下发的当前姿态。Yaw 的目标值使用 yaw 电机角度坐标系，位置反馈来自 yaw 电机主动上报的位置。

CAN 目标帧只刷新目标值，不刷新 RK 姿态反馈。若启用该可选业务模块，Roll/Pitch 姿态反馈超时时，mid 端应停止对应轴的电流输出。

## 10. 灯效区分

| 固件状态 | 灯效 |
| --- | --- |
| App 未配置 | 2 秒周期短闪一次 |
| App 已配置运行 | 2 秒周期三连短闪 |
| App 识别 | 按上位机指定频率闪烁 |
| Boot 待机 | 1Hz 闪烁 |
| Boot 升级中 | 5Hz 快闪 |

## 11. 过滤器说明

App CAN 初始化后接收：

```text
0x17E
0x100 + node_id
0x400 + node_id
```

未配置时 `node_id = 0x7E`，普通控制命令和 OTA 启动帧都会被协议层忽略，只处理发现、识别和配置业务 CAN ID 等管理命令。

mid 端若启用可选云台 CAN 业务模块，则额外接收：

```text
0x300
```

## 12. OTA 准入规则

OTA 只允许在电机已经配置唯一业务 CAN ID 后执行。上位机需要使用目标业务节点 ID 计算 OTA 帧 ID：

```text
OTA 控制 = 0x400 + node_id
OTA 响应 = 0x500 + node_id
```

多台未配置电机同时位于总线时，必须先通过 `0x27E` 收集 `UID32`，再通过 `0x61` 识别目标，最后通过 `0x62 + UID32` 配置唯一业务节点 ID。禁止直接对未配置管理节点 `0x7E` 发起 OTA，避免多台电机同时进入 Boot 并争用 `0x57E` 响应帧。

## 13. Flash 存储布局

64KB Flash 当前划分如下：

| 区域 | 起始地址 | 结束地址 | 大小 | 说明 |
| --- | --- | --- | --- | --- |
| Boot | `0x08000000` | `0x08004400` | 17KB | Bootloader 固件 |
| App | `0x08004400` | `0x0800F800` | 45KB | App 固件 |
| OTA 信息页 | `0x0800F800` | `0x0800FC00` | 1KB | OTA 标志、App 长度、App CRC16 |
| 用户信息页 | `0x0800FC00` | `0x08010000` | 1KB | CAN ID、零点、上报配置、FOC 基础配置 |

OTA 信息页独立于用户信息页。OTA 成功后 Boot 只更新 OTA 信息页，不擦写用户信息页；App 请求进入 Boot OTA 时也只更新 OTA 信息页的 OTA 标志。

## 14. OTA 业务头与校验

OTA 使用 YMODEM 传输，YMODEM 文件内容由 `128` 字节业务头和 App 镜像组成。Boot 不会把业务头写入 App Flash，App Flash 起始地址仍然写入真实 App 镜像。

业务头字段均使用小端格式：

| 偏移 | 长度 | 含义 |
| --- | --- | --- |
| `0x00` | 16 | 魔数，固定为 `MOTOR_DUCK_OTA\0\0` |
| `0x10` | 1 | 业务头版本，当前为 `1` |
| `0x11` | 1 | 业务头长度，当前为 `128` 字节 |
| `0x12` | 2 | 业务头 CRC16-CCITT，计算时本字段按 `0` 处理 |
| `0x14` | 4 | App 镜像长度，单位：字节 |
| `0x18` | 2 | App 镜像 CRC16-CCITT |
| `0x1A` | 102 | 保留，当前填 `0` |

CRC16-CCITT 使用多项式 `0x1021`，初值 `0x0000`。Boot 在收到首个数据包后校验业务头，业务头非法时不会擦除 App 区；写入完成后会读取 App Flash 重新计算 App CRC16，和业务头中的 App CRC16 不一致则 OTA 失败。

OTA 成功后，Boot 会把 `App 镜像长度` 和 `App CRC16` 保存到 OTA 信息页，并将 OTA 标志切回允许跳转 App。Boot 每次启动跳转 App 前都会先检查向量表合法性，再根据 OTA 信息页记录重新计算完整 App CRC16；校验失败会停留在 Boot。
