# CAN 协议说明

## 1. 概述

本项目使用标准帧 `CAN 2.0A`，采用“`ID + 命令码`”的通信形式。

- 标准帧长度：11 位 ID
- 数据长度：8 字节
- 数据字节序：小端
- 默认节点 ID：`0x10`

## 2. ID 规划

协议中的标准帧 ID 由基础 ID 与节点 ID 相加得到。

- 主机下发命令：`0x100 + node_id`
- 电机应答：`0x180 + node_id`
- 电机主动上报：`0x200 + node_id`

默认 `node_id = 0x10` 时：

- 主机下发命令 ID：`0x110`
- 电机应答 ID：`0x190`
- 电机主动上报 ID：`0x210`

## 3. 数据格式

所有命令帧的数据格式如下：

| 字节 | 含义 |
| --- | --- |
| Byte0 | 命令码 |
| Byte1~Byte7 | 命令参数 |

32 位整数参数统一使用小端编码。

## 4. 控制模式

控制模式定义如下：

- `0x00`：速度模式
- `0x01`：位置模式

## 5. 状态码

应答帧 `Byte1` 为状态码，定义如下：

- `0x00`：成功
- `0x01`：命令非法
- `0x02`：参数非法
- `0x03`：模式非法
- `0x04`：CAN 错误

## 6. PID 参数编号

速度环与位置环均使用相同的参数编号。

- `0x00`：`P`
- `0x01`：`I`
- `0x02`：`D`
- `0x03`：`I_ACC_MAX`
- `0x04`：`OUT_MAX`

其中：

- `P`、`I`、`D` 通过 `x1000` 的整数传输
- `I_ACC_MAX`、`OUT_MAX` 直接按内部整数值传输

## 7. 命令列表

### 7.1 设置运行模式

- 命令码：`0x01`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x01` |
| Byte1 | 使能：`0=停机`，`1=运行` |
| Byte2 | 控制模式：`0=速度模式`，`1=位置模式` |
| Byte3~Byte7 | 保留 |

示例：

```text
ID   = 0x110
DATA = 01 01 00 00 00 00 00 00
```

含义：进入速度模式并运行。

### 7.2 设置速度目标

- 命令码：`0x02`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x02` |
| Byte1~Byte4 | 目标速度，`int32`，单位 `mrad/s` |
| Byte5~Byte7 | 保留 |

示例：

```text
ID   = 0x110
DATA = 02 10 27 00 00 00 00 00
```

含义：设置目标速度为 `10000 mrad/s`。

### 7.3 设置位置目标

- 命令码：`0x03`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x03` |
| Byte1~Byte4 | 目标位置，`int32`，单位 `mrad` |
| Byte5~Byte7 | 保留 |

### 7.4 设置速度环参数

- 命令码：`0x10`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x10` |
| Byte1 | 参数编号 |
| Byte2~Byte5 | 参数值，`int32` |
| Byte6~Byte7 | 保留 |

示例：

```text
ID   = 0x110
DATA = 10 00 C4 09 00 00 00 00
```

含义：设置速度环 `P = 2.500`。

### 7.5 读取速度环参数

- 命令码：`0x11`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x11` |
| Byte1 | 参数编号 |
| Byte2~Byte7 | 保留 |

### 7.6 设置位置环参数

- 命令码：`0x12`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x12` |
| Byte1 | 参数编号 |
| Byte2~Byte5 | 参数值，`int32` |
| Byte6~Byte7 | 保留 |

### 7.7 读取位置环参数

- 命令码：`0x13`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x13` |
| Byte1 | 参数编号 |
| Byte2~Byte7 | 保留 |

### 7.8 设置 CAN 节点 ID

- 命令码：`0x20`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x20` |
| Byte1 | 新节点 ID |
| Byte2~Byte7 | 保留 |

### 7.9 读取 CAN 节点 ID

- 命令码：`0x21`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x21` |
| Byte1~Byte7 | 保留 |

### 7.10 零点校准

- 命令码：`0x30`
- 方向：主机 -> 电机

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x30` |
| Byte1~Byte7 | 保留 |

功能：

- 执行零点校准
- 保存零点到 Flash

### 7.11 读取零点

- 命令码：`0x31`
- 方向：主机 -> 电机

| 字节 | 含义 |

## 8. 电机应答帧

应答帧 ID 为：

```text
0x180 + node_id
```

应答帧格式如下：

| 字节 | 含义 |
| --- | --- |
| Byte0 | 回显命令码 |
| Byte1 | 状态码 |
| Byte2~Byte7 | 附加返回数据 |

## 9. 电机主动上报帧

主动上报帧 ID 为：

```text
0x200 + node_id
```

### 9.1 位置上报

- 命令码：`0x80`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x80` |
| Byte1~Byte4 | 电机位置，`int32`，单位 `mrad` |
| Byte5~Byte7 | 保留 |

### 9.2 速度上报

- 命令码：`0x81`

| 字节 | 含义 |
| --- | --- |
| Byte0 | `0x81` |
| Byte1~Byte4 | 电机速度，`int32`，单位 `mrad/s` |
| Byte5~Byte7 | 保留 |

## 10. 过滤器说明

协议初始化后，CAN 过滤器仅接收：

```text
0x100 + node_id
```

对应的主机命令帧。

默认节点 ID 为 `0x10` 时，仅接收 `0x110`。

## 11. 接收处理机制

CAN 接收中断回调函数仅负责：

- 从 FIFO0 读取报文
- 将报文压入协议接收队列

实际命令执行在主循环中的 `can_protocol_process()` 内完成。

这样可以避免以下耗时操作在中断上下文中执行：

- 零点校准中的延时
- Flash 擦写与保存
- CAN 过滤器重新配置
